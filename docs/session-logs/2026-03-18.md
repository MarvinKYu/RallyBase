#### v0.6.2 — Registration Lifecycle Automation

**Auto-open registration on publish**
- When a tournament is advanced to PUBLISHED, all DRAFT events are automatically moved to REGISTRATION_OPEN.

**Auto-close registration on start**
- When a tournament is advanced to IN_PROGRESS, all REGISTRATION_OPEN events are automatically moved to IN_PROGRESS.

**Auto-complete tournament on last event complete**
- When an event is manually advanced to COMPLETED and it's the last non-completed event, the tournament automatically advances to COMPLETED too.

**DRAFT tournament matches hidden from dashboard**
- Player dashboard no longer shows upcoming matches belonging to DRAFT tournaments.

#### v0.6.3 — Bracket Auto-Generation on Event Start

**Auto-generate bracket/schedule when event starts**
- When an event transitions to IN_PROGRESS (manually or via tournament-level cascade), the bracket (SE) or round-robin schedule is automatically generated if none exists yet and there are enough entrants (SE ≥ 2, RR ≥ 3).
- If conditions aren't met, the manual "Generate bracket" button remains available.
- Required a Vercel Cron job approach; implemented as part of the `advanceEventStatus` flow instead (no cron needed for the auto-generate trigger itself — cron would be needed for time-based auto-start, deferred to later).

### Known Issues / Notes
- `countNonCompletedMatches` counts ALL non-COMPLETED matches including PENDING ones with no players assigned (e.g. unplayed final-round slots in SE bracket). Events with TBD bracket slots will not auto-complete until every slot resolves — correct behavior.
- `tdVoidMatch` does not reverse event auto-complete: if a match is voided after the event auto-completed, the event stays COMPLETED. Matches existing behavior of the manual "Complete Event" button.
- Cron-based time-triggered auto-start (tournaments/events starting at their configured `startTime`) was deferred — requires Vercel Pro for sub-daily scheduling.
