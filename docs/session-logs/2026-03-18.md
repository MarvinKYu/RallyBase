### Known Issues / Notes
- `countNonCompletedMatches` counts ALL non-COMPLETED matches including PENDING ones with no players assigned (e.g. unplayed final-round slots in SE bracket). This means events with TBD bracket slots will not auto-complete until every slot resolves — which is correct behavior, since those matches need to be played.
- The `tdVoidMatch` path does not reverse the auto-complete: if a match is voided after the event auto-completes, the event stays COMPLETED. This is an edge case and matches the existing behavior of the manual "Complete Event" button (which is also not reversible from the UI).
