### Additional Details

#### Player vs TD navigation contract (established v0.7.6)
Two fully decoupled navigation paths — no shared pages with conditional TD/player content:
- **Player flow**: `/tournaments` ↔ `/tournaments/[id]` ↔ `/tournaments/[id]/events/[eventId]`
- **TD flow**: `/tournament-directors` ↔ `/tournaments/[id]/manage` ↔ `/tournaments/[id]/events/[eventId]/manage`

Bracket and standings pages are shared but context-aware: links from the TD flow pass `?from=manage`, and the back link on those pages reads the param to return to the correct destination.

#### Known issues / gotchas (from v0.6.5 implementation)
- **`countNonCompletedMatches` counts all non-COMPLETED matches**, including PENDING slots with no players assigned (e.g. unresolved final-round SE slots). Events with TBD bracket slots will not auto-complete until every slot resolves. This is correct behavior.
- **`tdVoidMatch` does not reverse event auto-complete**: if a match is voided after an event auto-completed, the event remains COMPLETED. Matches the behavior of the manual "Complete Event" button.
- **Cron-based time-triggered auto-start** (advancing tournaments/events to IN_PROGRESS at their configured `startTime`) was deferred — requires Vercel Pro for sub-daily scheduling. Bracket auto-gen on event start was implemented inside `advanceEventStatus` instead, requiring no cron for that trigger.
