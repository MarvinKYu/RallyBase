### Additional Details
- **Root cause of `advancersPerGroup` never saving**: Both `createEventAction` and `updateEventAction` were never updated to call `formData.get("advancersPerGroup")`. The field existed in the form, Zod schema, service, and repository — but the action layer never extracted it. Edit-event also didn't help because the same omission applied.
- **Auto-complete ordering**: Auto-complete must run AFTER SE auto-generate (not before), and must be skipped for group-phase matches in RR_TO_SE events — otherwise event completes when RR finishes but SE matches don't exist yet, causing `countNonCompletedMatches` to return 0 prematurely.
- **v0.14.3 deferred**: Placeholder SE bracket (computed cards showing "Group 1 Winner vs BYE" etc. before SE is generated) added to roadmap. Expected to also help with the event appearing COMPLETED during the gap between group stage finishing and SE being generated.

### Known Issues / Notes
- RR_TO_SE events created before v0.14.2 will have `advancersPerGroup = null` in production — fix via the edit event page after v0.14.2 deploys.
