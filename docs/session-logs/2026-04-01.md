## Session Log — 2026-04-01

### Objective
Fix the RR_TO_SE placeholder bracket bugs: premature event auto-complete, wrong placeholder seeding, and progress bar showing 100% before SE stage.

### Work Completed

#### v0.15.3 — RR_TO_SE placeholder bracket fixes
- **Auto-complete guard**: replaced `groupNumber`-based `isRRPhaseMatch` check with `checkEventComplete()` helper. For RR_TO_SE events, auto-complete now fires only when SE bracket exists (`countSEMatches > 0`) and is fully played (`countNonCompletedSEMatches === 0`). Removes fragile dependence on `match.groupNumber` which can be null on stale data.
- **`tryAutoGenerateSEStage`**: dropped `matchGroupNumber` parameter; now guards with `countSEMatches > 0` (SE already generated) instead of `matchGroupNumber === null`. Makes auto-generation resilient to null groupNumber on RR matches.
- **`tdDefaultMatch`**: added RR_TO_SE auto-complete guard (was completely missing — defaulting the last RR match would prematurely complete the event).
- **Placeholder bracket seeding**: replaced hand-rolled modular formula with `buildPlaceholderSeedInfo()` that calls `computeAdvancers` on synthetic standings. Correctly reproduces A=2 constrained half-zone runner-up placement.
- **Progress bar**: for RR_TO_SE events, replaced single bar with two bars — "Group stage progress" and "Bracket stage progress" (shows "Not started" until SE bracket generated).
- Added `countNonCompletedSEMatches` to `bracket.repository.ts`.

#### v0.15.4 — RR_TO_SE "View Standings" redirect fix
- Standings page guard `if (event.eventFormat !== "ROUND_ROBIN") redirect(...)` always redirected RR_TO_SE events. Fixed by also allowing `RR_TO_SE`. SE phase matches (groupNumber = null) filtered from the schedule so only the group stage appears.

#### v0.15.5 — Bracket stacking, centering, (D) indicator, UI polish
- **SF/Final stacking**: for brackets with `totalRounds ≥ 4` (16+ players), SF Left / Final / SF Right are stacked vertically in a single center column with vertical SVG connector lines. Right side absolutely positioned at `top: H`. Implemented via `StackedCenterColumn` component.
- **Bracket centering**: flat and stacked layouts both now center within the page using `flex justify-center`. Placeholder bracket also centered (`mx-auto w-fit`).
- **Default-win (D) indicator**: `MatchCard` and `StandingsSchedule` append `(D)` to winner name for `isDefault=true` matches. Footer status shows "default". Game win counts hidden for default wins.
- **"Start bracket"**: renamed the RR→SE SE stage button from "Generate bracket" to "Start bracket".
- **"View Groups"**: renamed "View Standings" → "View Groups" on manage page and event detail page.
- **4-column standings grid**: standings page for grouped events now uses `h-screen flex flex-col` with `grid-cols-4`, each group card scrollable. `StandingsSchedule` gained `compact` prop for denser rendering in group cards.

#### v0.15.6 — Stacked bracket layout fix
- **Root cause**: left side was at `position:absolute, top:0` and right side at `position:absolute, top:H` → asymmetric L-shape.
- **Fix**: replaced absolute layout with a simple flex row. Left and right columns now use `stackingFactor=2^round` (vs `2^(round-1)` in flat layout) so matches span `2H`. New `StackingConnector` draws two horizontal lines at `H/2` and `3H/2` (connecting QF to SF1/SF2). `BracketColumn` and `ForkConnector` gained optional `stackingFactor` prop.
