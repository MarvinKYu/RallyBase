#### v1.0.9 — Auto-regen on entrant add
- Root cause: `addEntrant` and `addBulkEntrantsAction` created event entries but never checked for a stale bracket/schedule. Unlike `tdRemoveEntrant` (fixed in v1.0.7), there was no regen step on add.
- Added `regenEventScheduleIfStale(eventId)`: checks `countMatchesByEventId > 0 && countProgressedMatches === 0`, then wipes and regenerates.
- `addEntrantAction` calls it after a successful single add; `addBulkEntrantsAction` calls it once after all adds complete.

### Additional Details (not in CHANGELOG)

**v1.1.0 implementation notes**

- The td-submit page already supported `?returnTo=` — no changes needed there.
