#### v1.0.9 — Auto-regen on entrant add
- Root cause: `addEntrant` (and `addBulkEntrantsAction`) created the event entry but never checked if a stale bracket/schedule existed. Unlike `tdRemoveEntrant` (fixed in v1.0.7), there was no regen step on add.
- Added `regenEventScheduleIfStale(eventId)` service function: checks `countMatchesByEventId > 0 && countProgressedMatches === 0`, then wipes and regenerates.
- `addEntrantAction` calls it after a successful single add.
- `addBulkEntrantsAction` calls it once after all adds complete (one regen, not N).
