## Session Log — 2026-04-15

### Objective
Bug fixes: auto-regen bracket/schedule when entrant added before any matches are played (counterpart to v1.0.7 removal fix).

### Work Completed

#### v1.0.9 — Auto-regen on entrant add
- Root cause: `addEntrant` (and `addBulkEntrantsAction`) created the event entry but never checked if a stale bracket/schedule existed. Unlike `tdRemoveEntrant` (fixed in v1.0.7), there was no regen step on add.
- Added `regenEventScheduleIfStale(eventId)` service function: checks `countMatchesByEventId > 0 && countProgressedMatches === 0`, then wipes and regenerates.
- `addEntrantAction` calls it after a successful single add.
- `addBulkEntrantsAction` calls it once after all adds complete (one regen, not N).

#### v1.1.0 — 3rd/4th place match
- Schema: added `Event.hasThirdPlaceMatch Boolean @default(false)` and `Match.isThirdPlaceMatch Boolean @default(false)`.
- Migration: `prisma/migrations/20260415_add_third_place_match/migration.sql` applied.
- `bracket.repository.ts`: added `findThirdPlaceMatch(eventId)`.
- `match.repository.ts`: `findMatchById` and `findPendingSubmissionByMatchId` now include `event.hasThirdPlaceMatch` and `nextMatch.isThirdPlaceMatch`.

- `match.service.ts`: `routeLoserToThirdPlace` helper populates the 3rd place match when an SF completes; `clearLoserFromThirdPlace` helper removes the player when an SF is voided; all four result paths (`confirmMatchResult`, `tdSubmitMatch`, `tdDefaultMatch`, `tdVoidMatch`) call the appropriate helper.
- `tournament.service.ts` / `tournament.actions.ts` / `tournament.repository.ts`: `createEvent` and `updateEvent` persist `hasThirdPlaceMatch` (SE-based formats only). Repository types updated.

- `bracket/page.tsx`: `isThirdPlaceMatch` matches separated before round-building; `FinalAndThirdColumn` component for small brackets; `StackedCenterColumn` extended with optional `thirdPlaceMatch` prop for large brackets; inline "Final" and "3rd/4th Place" labels added.

- `EventMatchList.tsx`, `ManageEventRightSection.tsx`, `RRtoSEMatchesList.tsx`: 3rd place match separated into its own "3rd/4th Place" section in all match list views.
