### Known Issues / Notes
- **Event status change UI missing**: there is no way for TDs to change an event from `DRAFT` to `REGISTRATION_OPEN` through the UI. Players cannot register for draft events. This is required before the registration flow is fully usable in production — planned for v0.6.0 (TD dashboard/publish workflow).
- **Timezone limitation**: `datetime-local` inputs are stored as UTC. Display via `toLocaleString()` converts to the browser's local timezone, which may confuse users. UTC labels added as a short-term mitigation. Proper timezone handling is a future improvement.
- **`prisma migrate dev` unavailable**: non-interactive terminal requires manually writing migration SQL and using `prisma db execute` to apply it. Use this pattern for all future schema migrations in this environment.

### Current State
v0.5.0–v0.5.2 are live on Vercel. The player registration overhaul is complete at the infrastructure level: the `/register` page, bulk signup, withdrawal logic, and start-time/deadline fields all work correctly. The main gap before this flow is fully usable end-to-end is a TD-facing way to set events to `REGISTRATION_OPEN` — the first item to address in v0.6.0. Next target is v0.6.0 (Tournament Lifecycle), which will include the publish/draft workflow, TD dashboard, event editing, and past tournaments category.
