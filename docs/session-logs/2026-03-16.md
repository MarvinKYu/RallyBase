#### v0.4.0 — Security, Integrity & Correctness
- **TD restrictions**: event creation, entrant addition, bracket generation locked to `createdByClerkId`; UI hides controls for non-TDs; server actions guard with auth check
- **Name uniqueness**: `@@unique([organizationId, name])` on Tournament, `@@unique([tournamentId, name])` on Event; P2002 caught and returned as field errors
- **Void AWAITING_CONFIRMATION**: `tdVoidMatch` and Void button extended to cover `AWAITING_CONFIRMATION` state
- **RR tiebreaker**: extracted `computeHeadToHead` + `compareTiebreakers` to `src/server/algorithms/round-robin-tiebreaker.ts`; standings sort uses H2H subset; tied players show `T-1`, `T-2` etc.; 9 unit tests added
- **4-digit verification code**: schema removes `@default(cuid())`; `match.service.ts` generates `Math.floor(Math.random()*10000).toString().padStart(4,"0")`; confirm form updated
- **Player search rating**: global search and add-entrant search show rating; context-sensitive category selection; "Unrated" fallback
- Migration: `prisma db push` + manually created `20260316000000_v0_4_0_name_uniqueness_and_4digit_code/migration.sql` + `migrate resolve --applied`

#### v0.4.1 — UI Polish
- **RR bracket redirect**: bracket page redirects to standings for ROUND_ROBIN events; generate schedule also redirects to standings
- **Score input UX**: `SubmitResultForm` rewritten with controlled `useState` inputs; focus clears field, blur resets to 0, failed submission retains values, invalid game row highlighted in red
- **Void button alignment**: form wrapper set to `flex items-center` in bracket and standings pages
- **Dashboard nav link**: added to `MobileNav` NAV_LINKS

#### v0.4.2 — Delete Event

- `deleteEvent` in service: ownership check

- `DeleteEventButton` component: same confirm-dialog + `useTransition` pattern as `DeleteTournamentButton`
- Button placed in event detail page header, visible to TD only
