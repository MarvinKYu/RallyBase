## Session Log — 2026-03-27

### Objective
Bracket redesign: two-sided layout with SVG connectors, seeding algorithm fix, and TD submit nav fix.

### Work Completed

#### v0.14.3 — Two-sided bracket, seeding fix, nav fix

**Seeding algorithm fix** (`bracket.ts`)
- Replaced linear `p vs bracketSize-p+1` formula with the standard recursive bracket seeding algorithm (`bracketSeedOrder`). Seeds 1 and 2 now always land on opposite halves for any bracket size, meeting only in the final. Previous behavior put seeds 1 and 2 both in the left half, causing them to meet in the semis.

**Bracket page redesign** (`bracket/page.tsx`)
- Two-sided layout: left half columns (R1→SF), centered Final, right half columns (SF→R1). Round labels mirrored on both sides.
- SVG connector lines: fork connectors (the ─┐/├─/└─ shape) between multi-match rounds, simple horizontal lines between 1-match rounds and the Final. Computed from known geometry constants (CARD_H, bracketSize).
- Match card updated: removed W/L labels, replaced with green-highlighted game-wins count for winner. Cleaner and consistent with reference design.
- Applies to both pure SE and RR_TO_SE hybrid events.

**TD submit nav fix** (`match.actions.ts`, `TdSubmitResultForm.tsx`, `td-submit/page.tsx`)
- `tdSubmitResultAction` and `tdDefaultMatchAction` now accept a `redirectTo` bound parameter.
- `td-submit/page.tsx` passes its already-computed `backHref` as `redirectTo` (bracket for SE matches, standings/manage for RR matches).
- SE bracket matches now return to the bracket page after TD submit or default win.

**Tests** (`tests/unit/bracket.test.ts`)
- Updated old seeding expectations to reflect correct algorithm.
- Added 8-player position tests and "seeds 1 and 2 always on opposite halves" property test.

#### v0.14.4 — Multi-select add entrants

**Multi-select add entrants** (`AddEntrantForm.tsx`, `tournament.actions.ts`, `manage/entrants/page.tsx`)
- `addBulkEntrantsAction`: accepts `playerProfileIds[]` from FormData, calls `addEntrant` per player, silently skips already-entered players, surfaces other errors.
- `AddEntrantForm` rewritten: checkboxes per player, single "Add N players" button (disabled when none selected), `useActionState` for error display.
- Entrants page now filters out already-entered players server-side before passing to `AddEntrantForm` — registered players disappear from the list after submission.

**Demo seed script** (`prisma/seed-extra-players.ts`)
- One-time script: 24 additional demo players with USATT ratings distributed 1000–2000.
- Run with `npx tsx prisma/seed-extra-players.ts`. Idempotent (skips already-existing players).

#### v0.14.5 — Entrants pagination fix; manage event state persistence

**Entrants pagination fix** (`player.service.ts`, `manage/entrants/page.tsx`)
- Added `excludeIds` option to `PlayerSearchOptions`; `searchPlayers` now filters before pagination so total/page counts are accurate.
- Entrants page passes `enteredIds` as `excludeIds` instead of filtering the fetched page client-side.

**Manage event state in URL** (`ManageEventRightSection.tsx`, `manage/page.tsx`)
- Selected sort (by group/round), phase (RR/SE), match page, and groups page now stored in URL search params (`sort`, `phase`, `mp`, `gp`) using `useSearchParams` + `router.replace`.

- Note: td-submit navigation still reset state — fully fixed in v0.14.7.

#### v0.14.6 — Placeholder SE bracket

**Placeholder SE bracket** (`bracket/page.tsx`, `bracket.ts`)
- Exported `bracketSeedOrder` from `bracket.ts`.
- When an RR_TO_SE event has group matches but no SE matches yet, the bracket page shows a full two-sided placeholder using the same layout and connectors as the real bracket.
- R1 slots display expected labels (e.g. "Group 1 — 1st", "Group 4 — 2nd", "BYE") computed from `bracketSeedOrder` + inter-group snake seeding. Higher rounds show "TBD".
- Falls back to the old "SE bracket not generated yet" message if the RR hasn't started yet.
- `PlaceholderCard` and `PlaceholderBracketColumn` components share geometry constants with the real bracket.
