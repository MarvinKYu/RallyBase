### Additional Details

**Seeding bug root cause (v0.14.10)**
`bracketSeedOrder(8) = [1,8,4,5,2,7,3,6]` pairs seed 1 vs seed 8 in R1. The old snake seeding assigned G1 runner-up → seed 8 (reversed for even ranks), so G1 winner (seed 1) and G1 runner-up (seed 8) were immediately paired. Fix: all ranks use ascending group order. Seeds 5–8 = G1–G4 runners-up. R1 pairs: 1v8(G1W/G4R), 4v5(G4W/G1R), 2v7(G2W/G3R), 3v6(G3W/G2R).

**State persistence pattern (v0.14.5 + v0.14.7)**
- v0.14.5 stored sort/phase/page in URL params — partially worked but td-submit redirect dropped them.
- v0.14.7 completed the fix: "Enter result" links embed `?returnTo=<encoded-current-url>` so the td-submit page always redirects back to the exact manage URL.

**Groups computation pattern**
- ManageEventRightSection: client-side useMemo over passed-in `matches` + `entries` props.
- Event detail page: identical logic computed server-side from Prisma auto-included scalar fields (`allMatchesRaw.groupNumber`, `event.eventEntries[].groupNumber`, `event.advancersPerGroup`). No extra DB queries needed.
