### Additional Details

- **Rating display in groups section**: shows current rating for the event's `ratingCategoryId`, not historical rating at draw time. No snapshot mechanism exists yet. Labeled "Rtg" rather than claiming historical accuracy. Historical rating preservation noted as a future feature in `feature-plans-shortlist.md`.
- **Multi-group event podium**: `getEventPodium` returns `{first: null, second: null}` for events with `groupSize` set — no cross-group ranking until v0.14.0 (RR → SE Hybrid).
- **Snake seeding confirmed**: future League format will use sequential blocks instead of snake distribution.
- **Group naming confirmed**: numeric only ("Group 1", "Group 2", …).
- **Invalid player count**: blocked with an error at schedule generation (e.g. "Not enough players for 2 valid groups with group size 4. Need at least 6 players (currently 5).").
