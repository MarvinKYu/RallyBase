- New `PlayerPagination` client component: reused across all three pages
- Public `/players`: default sort = Rating ↓ using USATT Singles; org filter → org's Singles; both filters → selected discipline

- TD entrants search: sort controls appear when query is active; sort rating uses event's `ratingCategoryId` (patched in v0.12.1)
- `searchPlayers` service updated to accept sort/pagination options and return `{ players, total, totalPages, page }`; in-memory sort handles rating (left-join on related field), lastName (last word of displayName), firstName (first word)
- Repository `searchProfiles`: removed hardcoded `take: 20` and `orderBy`
