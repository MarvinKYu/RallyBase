### Additional Details

- **Default rating is implicit**: No `PlayerRating` row is created when a player signs up. `applyRatingResult` falls back to `DEFAULT_RATING = 1500` via `?? DEFAULT_RATING` when no row exists. A player's first visible rating is already their post-match value. The admin initial-rating feature (v0.11.1) fills this gap by explicitly creating the row before any matches.
- **`isAuthorizedAsTD` pattern**: Single shared helper in `admin.service.ts` — returns `true` if the requesting user is the tournament creator, a platform admin, or an org admin for the tournament's org. Applied to all 9 TD-gated checks across `tournament.service.ts`, `tournament.actions.ts`, and `match.actions.ts`.
- **Platform admin assignment**: test_user_1 (player #1, `playerNumber = 1`) assigned PLATFORM_ADMIN role via Node script using Prisma upsert. No migration needed — `PLATFORM_ADMIN` was already in the `RoleName` enum.
- **OrgAdmin vs UserRole**: `OrgAdmin` is a separate table with `organizationId` scope. `UserRole` has no org scope and is used only for PLAYER and TOURNAMENT_DIRECTOR roles (and PLATFORM_ADMIN for the single admin account).
- **DB backfill for demo users** (gender/DOB for seeded test accounts) deferred — can now be done via `/admin/players` + initial rating assignment.
