#### v0.15.0 — RallyBase Rating System
- Added RallyBase org (slug: `rallybase`) with Singles discipline and "RallyBase Singles" rating category to `seed.ts`. Open to all authenticated users for tournament creation.
- Added `TournamentCreatorAllowlist` schema model + migration (`20260331000000`), paralleling `OrgAdmin`. Fields: `id`, `userId`, `organizationId`, `createdAt`, unique on `(userId, organizationId)`.
- New `src/server/algorithms/rating-algorithm.ts`: `RatingAlgorithm` interface + `getAlgorithmForOrg(orgSlug)` dispatcher. All orgs currently use the Elo adapter. Swap in custom RallyBase algo later by adding a case for `"rallybase"` slug.
- `rating.service.ts` uses `getAlgorithmForOrg` instead of importing `calculateMatchElo` directly. Org slug resolved from `ratingCategoryId` → org relation at match time.
- `canCreateTournamentInOrg(clerkId, orgId)` in `admin.service.ts`: returns true for platform admin, org admin, allowlist entry, or slug === `"rallybase"`.
- `getOrgsForTournamentCreation(clerkId)`: returns only orgs the user is authorized to create in. Used in `tournaments/new/page.tsx` (replaced the unfiltered `getOrganizations` call).
- `createTournament` service enforces `canCreateTournamentInOrg` before writing to DB.
- Admin page gated on `isAdminUser` (was `isPlatformAdmin`). `listOrgsForAdminPage` returns per-role filtered data. Platform admins see all orgs with Org Admins + Tournament Creators sections; org admins see only their orgs with Tournament Creators section only.
- `assignTournamentCreatorAction` / `removeTournamentCreatorAction` with org-scoped auth (platform admin or org admin of that org).
- `AssignTournamentCreatorForm` component for the admin page.
- Reverted an uncommitted `DEFAULT_RATING` change in `elo.ts` (1200 → 1500) that was breaking one unit test.
- 161 unit tests passing.
