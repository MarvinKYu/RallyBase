## Session Log ‚Äî 2026-03-13

### Objective

Stage 2 feature implementation: player search improvements, player self-signup, round-robin bracket format, TD/player view separation, and bracket UI spacing fix.

### Work Completed

**F5 ‚Äî Bracket UI Alignment Fix**
- Introduced `CARD_H = MATCH_H (80) + ACTIONS_H (24) = 104px` constant for all bracket spacing calculations.
- All match cards now render at a uniform 104px height; vertical spacing between cards is now correct across all rounds.

**F1 ‚Äî Player Search Improvements**
- Schema: added `playerNumber` (autoincrement, unique), `gender` (Gender enum), `birthDate` to `PlayerProfile`.
- Search supports name OR player number (`#42` syntax).
- URL param filters: organization, discipline/rating category, gender.
- Player number displayed on search results cards and profile page.
- Player's own profile page shows an **upcoming matches dashboard** (next matches they are entered in).

**F2 ‚Äî Player Self-Signup**
- Schema: added eligibility fields to `Event` ‚Äî `maxParticipants`, `minRating`, `maxRating`, `minAge`, `maxAge`.
- `checkEligibility()` pure function validates all conditions (capacity, rating range, age range).
- `SignUpButton` component: appears on event detail page when event is `REGISTRATION_OPEN` and the logged-in player is not yet entered.
- `signUpForEventAction` server action handles registration with eligibility check.
- `EventForm`: eligibility settings section (expandable) shown to tournament creators.
- `ProfileForm`: added gender and date-of-birth fields.

**F3 ‚Äî Round Robin Format**
- Schema: added `EventFormat` enum (`SINGLE_ELIMINATION`, `ROUND_ROBIN`); `Event.eventFormat` defaults to `SINGLE_ELIMINATION`.
- `buildRoundRobinSchedule()` ‚Äî circle-method algorithm (`src/server/algorithms/round-robin.ts`), supports 3‚Äì6 players.
- `generateBracket` branches on format: single-elimination uses existing seeded bracket logic; round-robin creates matches without advancement chain.
- Standings page at `/tournaments/[id]/events/[eventId]/standings` ‚Äî live W/L table + round-by-round schedule.
- `EventForm` has format selector.
- 11 new unit tests for round-robin algorithm (93 unit total).

**F4 ‚Äî TD/Player View Separation**
- `tdSubmitMatch`: TD records result directly via `/matches/[id]/td-submit`, no confirmation code required; runs Elo update immediately.
- `tdVoidMatch`: reverses rating transactions (restores `ratingBefore` values), deletes games/submissions, resets match to `PENDING`.
- Bracket page and standings page conditionally show **Enter result / Void** buttons for TDs vs. **Submit / Confirm** for players.
- Tournaments list page shows **"My tournaments"** section for entered players.
- Player profile shows upcoming match dashboard (own profile only).

**Schema Migration**

- New fields: `PlayerProfile.playerNumber`, `PlayerProfile.gender`, `PlayerProfile.birthDate`, `Event.eventFormat`, `Event.maxParticipants`, `Event.minRating`, `Event.maxRating`, `Event.minAge`, `Event.maxAge`.
