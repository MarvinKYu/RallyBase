#### v1.0.7 — Player removal bug fixes
- Diagnosed three root causes: (1) Pattern B action throwing on error → generic crash screen, (2) SE bye matches (status=COMPLETED, player2Id=null) being counted as "progressed" by `countProgressedMatches`, causing SE removal to always fail after bracket generation, (3) `deleteEventEntry` not cleaning up PENDING matches, leaving ghost matches visible and submittable after removal.
- `countProgressedMatches` now excludes structural bye matches via `OR [player2Id != null, isDefault = true]` filter.
- `tdRemoveEntrant`: updated error message; when removal is valid and a schedule exists, validates remaining count against format minimums, then deletes EventEntry + all matches and regenerates bracket/schedule with the updated player pool.
- `removeEntrantAction`: redirects with `?error=` instead of throwing, so error is visible to TD.
- `ManageEntrantsPage`: reads `?error=` from searchParams and renders a red error banner.
